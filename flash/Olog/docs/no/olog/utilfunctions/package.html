breakPoint()function 
public function breakPoint(... args):void

Sets a virtual break point. Code execution is not halted, but the class, function name and line number of the call to breakPoint is written to the log window along with introspection of any argument passed to it.

Parameters

... args — Values to inspect at break point. There are two ways to use this argument:
  1. Use it just like a standard trace and pass any properties you want the values of to be displayed after the breakpoint
  2. Pass an object reference as the first argument and string property names after that. These property names will be concidered public properties of the object in the first argument and traced.
  3. Pass an object reference as the first argument and " as the second argument to invoke a full discription of the object passed as the first argument

otrace()function 
public function otrace(message:*, level:uint = 1, origin:* = null):void

Shorthand for Olog.trace(). Outputs content to log. The message argument can be anything from basic strings to complext objects. Some objects types will be parsed further to display informal output about them. Use the level argument to specify a severity level with corresponding text color.

Parameters

message:* — Any type object to display as message in log
 
level:uint (default = 1) — Severity level
  • 0: Dimmed, use for unimportant/highly repetative content
  • 1: Default, white
  • 2: Orange, typicaly used for warnings
  • 3: Red, typically used for errors. Olog uses this level automatically when message argument is Error or ErrorEvent
  • 4: Green, typically indicates success of some sort
  • 5: Blue, for notable events and default when message argument is of type Event
 
origin:* (default = null) — A String or object specifying where in the application the message originated. Note: This parameter should be passed as a String if Olog is to be included in the release build for optimum performance.
